# Fraud Risk Scoring Module

This module combines image forgery scores and ICD verifier outputs to produce a final fraud risk percentage for insurance claims.

Structure
```
fraud_risk_module/
├── synthetic_data_generator.py  # generates synthetic dataset for academic demo
├── feature_builder.py           # load dataset and build features for training
├── train_xgboost.py             # trains XGBoost classifier and saves model
├── predict_risk.py              # loads model and produces risk JSON + explanation
├── models/
│   └── fraud_model.pkl          # trained model (created by train script)
└── README.md
```

Key points
- Labels are synthetic and generated by rule-based heuristics for academic demonstration only.
- The model is an `XGBClassifier` (no deep learning) to keep inference lightweight and explainable.
- For deployment, integrate `predict_risk.predict_risk()` with upstream data ingestion and ensure features are normalized consistently.

Usage examples
1. Generate synthetic data:
```
python synthetic_data_generator.py --n 2000 --out data/synthetic.csv
```
2. Train model:
```
python train_xgboost.py
```
3. Predict risk (interactive):
```
python predict_risk.py --interactive
```

Ethics and limitations
- The synthetic labels are not real fraud labels and should not be used for operational decisions.
- This module is for academic demonstration and unit testing of the end-to-end pipeline only.
